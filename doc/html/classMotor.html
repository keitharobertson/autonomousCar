void Motor::init_device ( )
virtual

Virtual function to initialize the actuator device.

Defined for each actuator, the init_device function is responsible for setting the actuator up for control by the mechanical control task. For example in the case of the motor subsystem, the init_device task will set up the PWM, initialize the system variables, and perform other tasks necessary before the motor can be controlled.

void Motor::mech_command ( char *  value)
virtual

The mechanical command function issues commands directly to hardware.

The mechanical command function is typically called from the mech_control function and will issue the actual commands to hardware necessary to control the actuator. For example, for the motor subsystem, this function will write to PWM.

void * Motor::read_data ( int  command)
virtual

Reads in data from message to the actuator.

Will read data into appropriate data type and then cast to a void* and return. The type of data received depends on the message command. For example if trying to set the motor pwm duty cycle, a three byte char* is expected. This function will read the data in from standard in (using cin) into an appropriate datatype for the given command and will return the data to the system message handler for further processing.
